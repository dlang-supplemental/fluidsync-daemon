The **FluidSync GUI Daemon** (`fluidsync-daemon`) is a specialized telemetry and orchestration service designed to provide real-time temporal context to visual user interface applications. It acts as a system-wide "heartbeat" and budget provider, ensuring that GUI applications can maintain high-fidelity responsiveness and eliminate "ghosting" or stutter during heavy system load.

toc::[]

== 💡 The Core Problem: OS Resizing Artifacts
When you resize a window in Windows, the Desktop Window Manager (DWM) often "stretches" the existing window backbuffer while waiting for the application to redraw. This creates a "ghosting" effect where the UI appears to lag behind the cursor.

Fixing this requires intercepting `WM_SIZING` messages for synchronous redraws. However, doing so in a single-threaded UI loop can lead to massive frame drops if the UI layout is complex. `fluidsync-daemon` solves this by providing a high-performance, zero-latency telemetry buffer that tells the UI exactly how much time it has to draw before it must "cull" details to maintain the frame rate.

== 🚀 Quick Start (Tutorial)
_Note: The project is currently in the architectural phase. Real-world implementation is ongoing._

1. **Clone the repository:**
[source,powershell]
----
git clone https://github.com/dev-centr/fluidsync-daemon.git
----

2. **Build the daemon (requires D compiler and `dub`):**
[source,powershell]
----
dub build
----

3. **Run the daemon:**
[source,powershell]
----
./fluidsync-daemon.exe
----
