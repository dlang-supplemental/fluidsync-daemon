== What Is This?

When you resize some apps on Windows, Mac, or Linux, the window border moves fast but the stuff inside the window looks smeared, stretched, or lags. These are artifacts from the redraw process. They catch your attention and distract from the interaction.

The *FluidSync GUI Daemon* (+fluidsync-daemon+) is a tiny background helper that tells apps:

* "Here is how fast your computer is right now."
* "Here is how much time you really have to draw each frame."

With that information, apps can choose to:

* Draw the *simple* version of the screen while you are dragging the window.
* Draw the *fancy* detailed version after you stop dragging.

Result: the app feels solid and glued to your mouse instead of laggy and stretchy.

== How It Works (Simplified)

* +fluidsync-daemon+ watches the system in the background (CPU, frame time, etc.).
* It writes a tiny summary (a few numbers) into shared memory.
* Apps read those numbers very quickly (no heavy system calls).
* If things look "slow" or "busy", the app switches into a cheaper drawing mode so it does not stutter.

== Who Should Care?

You care about this project if:

* You build desktop apps (for example with +dlangui+).
* Your app has complex layouts (big tables, many widgets, lots of text).
* You want resizing and heavy UI actions to feel smooth even on a busy machine.

If you just want to *use* the daemon, you only need the quick start section.
If you want to *integrate* it with your UI framework, see the docs listed below.

== Quick Start (Very Simple)

_Note: The project is still in the architectural / early implementation phase. Things may change._

1. Clone the repo:

[source,powershell]
----
git clone https://github.com/dlang-supplemental/fluidsync-daemon.git
cd fluidsync-daemon
----

2. Build the daemon (needs a D compiler and +dub+):

[source,powershell]
----
dub build
----

3. Run the daemon:

[source,powershell]
----
./fluidsync-daemon.exe
----

At this point, the process should be running in the background and exposing its shared memory and IPC endpoints.
